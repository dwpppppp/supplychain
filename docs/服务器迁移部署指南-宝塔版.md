# 供应链导源课程平台服务器迁移部署指南（Ubuntu 22 + 宝塔面板）

这份文档是给“会操作电脑，但不一定熟悉服务器开发”的人看的。目标是把本地项目迁移到一台全新的 Ubuntu 22 服务器上，并用宝塔面板管理。

项目上线后大致会变成这样：

- 用户访问：`https://你的域名`
- 前端页面：Next.js，运行在服务器本机 `3030` 端口
- 后端接口：FastAPI，运行在服务器本机 `8000` 端口
- 数据库：MySQL
- 缓存：Redis
- 反向代理：Nginx，把外部访问转发到前端和后端

---

## 一、需要准备什么

### 1. 服务器

建议最低配置：

- 系统：Ubuntu 22.04
- CPU：2 核或以上
- 内存：4GB 起步，8GB 更稳
- 磁盘：40GB 起步

如果课程素材、PDF、图片越来越多，磁盘建议更大。

### 2. 域名

建议准备一个域名，例如：

```text
sourcing.example.com
```

如果暂时没有域名，也可以先用服务器 IP 测试，但正式使用建议绑定域名并开 HTTPS。

### 3. 本地项目文件

需要上传整个项目目录：

```text
F:\supplychain-claude
```

但不建议上传这些目录：

```text
backend/venv
frontend/web/node_modules
frontend/web/.next
```

这些目录到服务器后重新安装/构建即可。

### 4. 数据库备份

你本地 MySQL 里的数据也需要迁移，尤其是：

- 用户账号
- 章节表 `chapters`
- 课程图谱 `knowledge_nodes`、`knowledge_edges`
- 实训案例相关表
- 学生学习进度、笔记、聊天记录等

可以用 Navicat、DBeaver、宝塔数据库导入，或者用 `mysqldump` 导出 `.sql` 文件。

## 二、宝塔面板要装哪些软件

新服务器先安装宝塔面板，然后在宝塔的软件商店安装下面这些。

### 必装

| 软件 | 建议版本 | 用途 |
|---|---:|---|
| Nginx | 1.22+ / 1.24+ | 网站入口、反向代理 |
| MySQL | 8.0 优先，5.7 也可 | 存储业务数据 |
| Redis | 6.x / 7.x | 登录状态、限流、缓存 |
| Python | 3.10+ | 后端运行环境，Ubuntu 22 默认就是 3.10 |
| Node.js | 20.9 以上 | 前端 Next.js 16 需要 |
| PM2 管理器 | 最新 | 守护前端 Node 服务 |

### 可选但推荐

| 软件 | 用途 |
|---|---|
| Python 项目管理器 | 宝塔里更方便启动 FastAPI |
| Supervisor 管理器 | 如果不用 Python 项目管理器，可用它守护后端 |
| phpMyAdmin | 方便看数据库 |
| 宝塔 SSL / Let's Encrypt | 给域名开 HTTPS |

## 三、推荐目录结构

建议把项目放到：

```text
/www/wwwroot/supplychain-claude
```

最后服务器目录大概是：

```text
/www/wwwroot/supplychain-claude
├── backend
│   ├── app
│   ├── static
│   ├── sql
│   ├── main.py
│   ├── requirements.txt
│   └── .env
└── frontend
    └── web
        ├── app
        ├── components
        ├── features
        ├── lib
        ├── package.json
        └── .env.local
```

注意：`backend/static` 很重要，里面有课程 HTML、PDF、实训图片、供应商图片等素材，必须上传。

## 四、创建数据库

在宝塔面板：

1. 打开“数据库”
2. 新建数据库
3. 推荐填写：

```text
数据库名：supplychain
用户名：supplychain
密码：生成一个强密码
编码：utf8mb4
```

然后把本地导出的 `.sql` 数据库备份导入进去。

如果还没有完整数据库备份，也至少要导入你已经整理好的章节、图谱、实训相关数据。后端启动时会自动建缺失的表，但不会自动把你本地已经编辑好的课程章节内容全部补进去。

## 五、配置后端环境变量

后端配置文件在：

```text
/www/wwwroot/supplychain-claude/backend/.env
```

服务器上建议这样配置：

```env
DEBUG=false

DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=supplychain
DB_USER=supplychain
DB_PASSWORD=你的数据库密码

REDIS_HOST=127.0.0.1
REDIS_PORT=6379
REDIS_DB=0
# REDIS_PASSWORD=如果 Redis 设置了密码就填

JWT_SECRET_KEY=请换成一串很长的随机字符串
ACCESS_TOKEN_EXPIRE_MINUTES=60
REFRESH_TOKEN_EXPIRE_DAYS=30

DASHSCOPE_API_KEY=你的阿里云 DashScope Key
LLM_MODEL=qwen-plus
TTS_MODEL=qwen3-tts-flash-realtime
TTS_VOICE=longxiaochun

BAILIAN_WORKSPACE_ID=你的百炼工作空间 ID
BAILIAN_INDEX_ID=你的百炼知识库索引 ID

OSS_ACCESS_KEY_ID=你的 OSS AccessKey
OSS_ACCESS_KEY_SECRET=你的 OSS Secret
OSS_BUCKET_NAME=你的 OSS Bucket
OSS_ENDPOINT=oss-cn-beijing.aliyuncs.com
# OSS_CDN_DOMAIN=如果有 CDN 域名再填

CORS_ORIGINS=["https://你的域名"]
```

如果你先用 IP 测试，可以临时写：

```env
CORS_ORIGINS=["http://服务器IP"]
```

或者开发测试阶段先保持：

```env
CORS_ORIGINS=["*"]
```

正式上线建议改成具体域名。

## 六、安装并启动后端

进入后端目录：

```bash
cd /www/wwwroot/supplychain-claude/backend
```

创建 Python 虚拟环境：

```bash
python3 -m venv venv
```

启用虚拟环境：

```bash
source venv/bin/activate
```

安装依赖：

```bash
pip install -r requirements.txt
```

测试启动：

```bash
python main.py
```

看到类似下面内容就说明后端启动成功：

```text
Uvicorn running on http://0.0.0.0:8000
Database ready.
```

然后浏览器访问：

```text
http://服务器IP:8000/docs
```

如果能看到接口文档，后端正常。

### 后端正式守护方式

测试没问题后，不要一直靠 SSH 窗口运行。可以用宝塔的“Python 项目管理器”或“Supervisor 管理器”。

推荐启动命令：

```bash
/www/wwwroot/supplychain-claude/backend/venv/bin/python /www/wwwroot/supplychain-claude/backend/main.py
```

运行目录填写：

```text
/www/wwwroot/supplychain-claude/backend
```

端口：

```text
8000
```

## 七、配置前端环境变量

前端配置文件在：

```text
/www/wwwroot/supplychain-claude/frontend/web/.env.local
```

如果使用域名，填写：

```env
NEXT_PUBLIC_API_BASE_URL=https://你的域名/api/v1
```

如果先用 IP 测试，填写：

```env
NEXT_PUBLIC_API_BASE_URL=http://服务器IP/api/v1
```

注意：这里不要写 `127.0.0.1`。  
因为浏览器里的 `127.0.0.1` 指的是访问者自己的电脑，不是服务器。

## 九、配置 Nginx 网站

在宝塔面板：

1. 网站
2. 添加站点
3. 域名填写：`你的域名`
4. 根目录可以先选：`/www/wwwroot/supplychain-claude/frontend/web`
5. PHP 选择：纯静态

然后进入站点设置，找到“配置文件”，把反向代理配置改成类似下面这样：

```nginx
server {
    listen 80;
    server_name 你的域名;

    client_max_body_size 50m;

    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /static/ {
        proxy_pass http://127.0.0.1:8000/static/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location / {
        proxy_pass http://127.0.0.1:3030;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

保存后重载 Nginx。

如果开了 HTTPS，宝塔会自动多生成 SSL 配置，核心仍然是：

- `/` 转发到前端 `3030`
- `/api/` 转发到后端 `8000`
- `/static/` 转发到后端静态文件

## 十、开放哪些端口

正式使用时，公网只需要开放：

```text
80
443
```

下面这些端口建议不要对公网开放，只让服务器内部访问：

```text
8000 后端
3030 前端
3306 MySQL
6379 Redis
```

也就是说，用户只访问域名，Nginx 在服务器内部转发到 3030 和 8000。

## 十一、上线后检查清单

按顺序检查：

### 1. 后端健康检查

访问：

```text
https://你的域名/api/v1/system/health
```

如果这个路径不可用，也可以访问：

```text
https://你的域名/docs
```

或：

```text
https://你的域名/
```

如果前端占用了 `/`，后端根路径不用暴露也没关系。

### 2. 前端页面

访问：

```text
https://你的域名
```

检查：

- 首页能打开
- 登录弹窗能打开
- 登录后右上角状态正确
- 教学导览能显示知识图谱
- 课程文档能打开章节
- 模拟实训能进入案例
- AI 助学能正常请求

### 3. 静态素材

重点检查：

```text
https://你的域名/static/course-html/chapter-01.html
https://你的域名/static/training/training1.png
```

如果打不开，说明 `/static/` 代理或 `backend/static` 上传有问题。

### 4. 数据是否完整

检查：

- 左侧章节是不是 10 章都在
- 第 1-10 章课程文档是否能打开
- 实训案例是否有草莓、电机、危险品等案例
- 管理后台 `/training/admin` 是否能打开

管理后台需要 `teacher` 或 `admin` 角色账号。

## 十二、常见问题

### 1. 页面能打开，但接口 Request failed

优先检查：

- 前端 `.env.local` 里的 `NEXT_PUBLIC_API_BASE_URL` 是否是服务器域名，不要是 `127.0.0.1`
- 后端是否在运行
- Nginx `/api/` 是否正确代理到 `127.0.0.1:8000`
- 后端 `.env` 的 `CORS_ORIGINS` 是否允许当前域名

### 2. 登录后刷新又异常

检查：

- `JWT_SECRET_KEY` 是否固定，不要每次重启变化
- 前端访问域名是否统一，不要一会儿 IP、一会儿域名
- 浏览器是否拦截 Cookie 或本地存储

### 3. 课程图片、PDF、HTML 打不开

检查：

- `backend/static` 是否完整上传
- Nginx 是否有 `/static/` 代理
- 后端 `main.py` 是否正常挂载 `/static`

### 4. AI 助学不能用

检查：

- `DASHSCOPE_API_KEY` 是否填写
- 服务器能否访问外网
- 阿里云账号额度是否正常
- 百炼 `BAILIAN_WORKSPACE_ID` 和 `BAILIAN_INDEX_ID` 是否正确

### 5. 实训管理后台打不开

检查：

- 当前账号 `users.role` 是否是 `teacher` 或 `admin`
- 后端管理接口是否正常：

```text
GET /api/v1/training/admin/cases
```

### 6. MySQL 连接失败

检查：

- `.env` 里的数据库名、用户、密码
- 宝塔 MySQL 是否运行
- 数据库用户是否有权限访问 `supplychain`
- 密码里如果有特殊符号，建议换成数字、大小写字母组合，避免 URL 编码问题

## 十三、推荐的上线顺序

最稳的顺序是：

1. 安装宝塔
2. 安装 Nginx、MySQL、Redis、Node.js、Python
3. 创建数据库
4. 导入本地数据库备份
5. 上传项目文件
6. 配置后端 `.env`
7. 启动后端，确认 `8000/docs` 可访问
8. 配置前端 `.env.local`
9. `npm install`
10. `npm run build`
11. 启动前端，确认 `3030` 可访问
12. 配置 Nginx 域名反向代理
13. 开 HTTPS
14. 用域名完整测试登录、课程、AI、实训、后台

## 十四、备份建议

上线后建议每天自动备份：

- MySQL 数据库

- 后端 `.env`
- 前端 `.env.local`

`.env` 文件里有密钥，不要公开上传到代码仓库。

## 十五、这套项目最关键的几个点

记住这几个点，排查问题会快很多：

- 前端端口：`3030`
- 后端端口：`8000`
- 后端接口前缀：`/api/v1`
- 后端静态资源：`/static`
- 数据库：MySQL
- 缓存：Redis
- 前端生产启动：`npm run start`
- 后端生产启动：`python main.py`
- 前端 `.env.local` 不能写 `127.0.0.1`，要写域名或服务器 IP

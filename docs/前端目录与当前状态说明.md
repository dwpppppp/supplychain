## 3. 当前目录按什么思路组织

当前前端不是按“纯组件库”来拆，而是以 **页面域 + 功能域** 为主：

- `app/`：负责路由入口、页面装配、布局壳子
- `features/`：负责每个业务域自己的 API、组件、hooks、类型
- `lib/`：负责跨业务共享的基础能力

这样的好处是：

- 页面边界清楚，后续 5 个主要页面可以继续独立演进
- API、状态、组件都更容易按业务聚合，不容易重新混成旧前端那种“全局耦合”
- 后续如果课程页、实训页体量起来，也可以继续在各自 `features/*` 下扩展

## 7. 目前已有的前端机制

这一部分是最关键的，后续继续开发时，尽量都基于这些机制扩展，而不是重新散着写。

### 7.1 根布局机制

当前根布局负责：

- 注入全局字体
- 注入全局样式
- 包裹 `AuthProvider`

这意味着后续任何页面都可以直接使用全局认证状态，而不需要再单独在页面里重复写一套登录检测。

### 7.2 统一认证机制

当前认证机制包括：

- 登录
- 注册
- 注册后自动登录
- 本地 `localStorage` 持久化
- `sessionChanged` 事件广播
- 顶部登录状态同步
- 任意页面唤起登录弹窗

这套机制已经是全局统一入口，后续不要再在业务页面里自己写散的登录表单，优先复用：

- `features/auth/components/auth-provider.tsx`
- `features/auth/components/auth-access-panel.tsx`
- `features/auth/lib/session.ts`

### 7.3 统一请求机制

当前所有普通接口请求，已经统一通过：

- `lib/api/http.ts`

已有能力：

- 自动拼接 `API_BASE_URL`
- 自动传 `Authorization`
- 自动解析后端 `{ code, message, data }`
- 非 0 code 时统一抛 `ApiRequestError`

这套机制适合继续作为课程页、实训页、图谱页的基础请求入口。

### 7.4 API 地址与前端常量机制

当前已经统一收口到：

- `lib/constants/api.ts`

已管理的内容：

- `NEXT_PUBLIC_API_BASE_URL`
- 默认后端地址
- 本地 storage key
- 全局 session 事件名

### 7.5 页面分组机制

当前路由已经按路由组分开：

- `(auth)`：认证域
- `(platform)`：平台内业务页

这样后面如果再加“独立登录页样式”和“平台内统一头部布局”，边界会一直很清楚。

### 7.6 平台壳子机制

`app/(platform)/layout.tsx` 目前承接：

- 品牌标题
- 顶部导航
- 顶部登录状态

## 10. 2026-05-01 更新：教学导览知识图谱当前状态

教学导览页的知识图谱已经从占位状态推进为可交互图谱。当前主要实现集中在：

- `frontend/web/app/(platform)/overview/page.tsx`
- `frontend/web/features/overview/api/chapters.ts`
- `frontend/web/app/globals.css`

### 10.1 已接入的数据接口

| 前端能力 | API 封装 | 后端接口 |
| --- | --- | --- |
| 左侧章节树 | `fetchChapters()` | `GET /api/v1/course/chapters` |
| 知识图谱节点和关系 | `fetchKnowledgeGraph()` | `GET /api/v1/course/knowledge-graph` |
| 教学导览访问埋点 | `trackOverviewView()` | `POST /api/v1/analytics/track` |

### 10.2 当前图谱交互能力

- 使用 SVG 渲染知识图谱节点、边、箭头 marker 和节点标题。
- 节点标题完整展示在节点下方，按字符自动换行，不再截断到圆内。
- 支持鼠标滚轮缩放、拖拽平移、右下角 `+ / 重置 / -` 控件。
- 点击节点会打开右侧详情面板，展示节点类型、标题、描述和相关关系。
- hover/选中时会高亮相关边和关联节点，非关联节点降低透明度。
- 点击节点时保留一次性触感反馈：弹性缩放、颜色短暂加深、圆形涟漪。
- 图谱首次加载时有整体入场动画：边线绘制、节点错峰弹出、标题浮现。
- 节点采用局部坐标结构：外层 `<g transform="translate(x,y)">` 定位，内部圆心使用 `cx=0/cy=0`，避免点击缩放时节点漂移。

### 10.3 当前图谱布局和视觉策略

- 使用前端 `runSimulation()` 做一次性力导向布局，不在交互中重新跑模拟。
- 已加入更强排斥力、较宽节点间距和 collision pass，降低节点和标题重叠。
- canvas 容器使用 flex 布局和 `overflow: hidden`，SVG 内容不会溢出到底部图例区域。
- 正式图谱 SVG 已覆盖首页 teaser 的 `max-height/padding` 样式污染，画布可占满可用高度。
- 当前节点配色：
  - `chapter`: `#1e3a5f`
  - `concept`: `#3a6ea5`
  - `method`: `#6b9cc9`
  - `step`: `#a8b8a0`
  - `tool`: `#b8a888`
  - `metric`: `#c47b6c`
- 连线统一使用 `#c5cdd6`，画布背景为 `#f7f9fc`，节点统一白色描边。

### 10.4 当前保留的注意事项

- 点击节点的持续呼吸、持续放大、关联节点弹跳等持续动效已移除；后续点击效果如需调整，只改 `handleNodeClick()` 和 `.click-ring`。
- 整体图谱入场动画只在 `overview-graph-stage.is-entering` 期间触发，避免 hover 时重播。
- 不要把 `preserveAspectRatio` 改回 `none`，否则节点圆会变形。
- 不要把节点圆重新改成 `cx=p.x/cy=p.y` 再直接 scale，容易造成 SVG 缩放原点问题。
- 不要修改 `runSimulation()` 的方向时同时引入持续布局动画；当前图谱是“加载时布局一次，交互只做 pan/zoom/selection”。

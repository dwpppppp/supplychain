# 任务

你是一个知识图谱构建助手。我会给你一份教材某一章的 PDF 文本内容,你需要从中提取知识点节点和它们之间的关系,生成可直接执行的 MySQL INSERT SQL。

# 数据库 Schema

## 表 1: knowledge_nodes(知识点节点)

```sql
CREATE TABLE knowledge_nodes (
  id varchar(32) NOT NULL,           -- 主键,字符串,格式见下方命名规则
  title varchar(128) NOT NULL,        -- 简洁标题,5-15字
  description text,                   -- 一句话描述,30-80字
  icon varchar(64) DEFAULT NULL,      -- 留空字符串 ''
  node_type varchar(32) NOT NULL,     -- 节点类型,见下方枚举
  chapter_id bigint DEFAULT NULL,     -- 关联章节id,见下方映射
  sort_order int NOT NULL,            -- 章内排序,从1递增
  PRIMARY KEY (id)
);
```

## 表 2: knowledge_edges(知识点关系边)

```sql
CREATE TABLE knowledge_edges (
  id bigint AUTO_INCREMENT,
  source_node_id varchar(32) NOT NULL,  -- 源节点id
  target_node_id varchar(32) NOT NULL,  -- 目标节点id
  relation_type enum('contains','prerequisite','related','applied_in') NOT NULL
);
```

# 命名规则

## 节点 ID 格式
- 格式: `kn_ch{章号}_{三位序号}`
- 例: `kn_ch1_001`, `kn_ch2_015`, `kn_ch10_007`
- 序号从 001 开始,章内递增,不要跳号

## node_type 枚举(必须从中选一个)
- `chapter`:章级根节点,每章只有 1 个,统领全章
- `concept`:核心概念(如"供应源搜寻"、"供应商评估")
- `method`:方法论或分类(如"战略 vs 战术"、"10C 模型")
- `step`:流程步骤(如"识别需求"、"市场分析")
- `tool`:分析工具(如"帕累托原理"、"Kraljic 矩阵")
- `metric`:量化指标(如"KPI 价格"、"质量合格率")

## relation_type 枚举(必须从中选一个)
- `contains`:包含关系。父概念包含子概念,父类包含子类。最常用。
- `prerequisite`:前置关系。学 A 之前必须先理解 B。比如"市场分析"前置于"供应商资格预审"。
- `related`:相关关系。两个概念有联系但非包含或前置。比如"帕累托原理"与"Kraljic 矩阵"都是分析工具。
- `applied_in`:应用关系。某个工具/方法应用于某个场景或步骤。比如"帕累托原理"应用于"供应商分类"。

# 章节 ID 映射(根据我提供的章号填写 chapter_id)

每章的章级根节点(node_type='chapter')关联到该章的章 id;
每章下面的具体知识点关联到对应小节的 id。

我会在每次任务开始时告诉你本章的章节 id 映射,例如:

```

```

如果某个知识点跨多个小节(比如总览性的概念),关联到章级 id;
如果某个知识点明确属于某一节,关联到该节的 id。

# 输出要求

## 数量参考
- 节点数量:每章 15-25 个,过少则覆盖不全,过多则过于碎片
- 边数量:每个节点平均 2-3 条边,即 30-60 条
- chapter 节点:每章 1 个,作为根节点
- 其他节点:concept、method、step、tool、metric 按内容自然分配,不强求均衡

## 关系构建原则
1. **章根节点必须 contains 所有顶层概念**(章节最重要的几个概念)
2. **流程类内容用 prerequisite 串起来**(第一步 prerequisite 第二步,以此类推)
3. **工具应用要有 applied_in 边**(如帕累托 applied_in 供应商分类)
4. **同类概念之间用 related**(如战略 sourcing related 战术 sourcing)
5. **避免冗余边**:如果 A contains B 且 B contains C,不要再加 A contains C

## 描述写作要求
- 用陈述句,不用问句
- 不要复制 PDF 原文,要用自己的话总结
- 抓住该概念**最关键的判别特征**,不要泛泛而谈
- 反例:"供应商评估是一个重要的活动" ❌ 太空
- 正例:"对供应商能力、质量、交付、财务和服务进行综合判断,常用 10C 模型" ✓ 具体
